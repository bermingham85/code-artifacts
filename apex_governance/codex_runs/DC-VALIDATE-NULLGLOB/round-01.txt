**Why:** The user pushed hard on "Codex is the primary gate" and "do not bounce decisions straight to operator when Codex can gate." Combined with the participant protocol stop-conditions (Supabase/Notion/GitHub writes need human approval), this loop balances autonomy with safety and avoids both extremes.

**How to apply:**

1. **Diagnose** — read evidence first (queue contract, ai_audit history, repo state, live SQL). Write findings to `audit/<date>/task_<id>/diagnosis.md`. Distinguish verified facts from assumptions.
2. **Propose** — concrete operation with exact SQL/code/PowerShell. Write `migration_decision.md` or equivalent. List residual risks honestly.
3. **Codex-gate** — `codex exec -s read-only -` (sandbox=read-only, stdin prompt). Inline the SQL/code because Codex's Windows sandbox can't reliably read files via pwsh (`windows sandbox: spawn setup refresh`). Return JSON verdict only.
4. **Revise on NEEDS_REVISION** — address each `required_changes[]` item; re-gate. Don't argue.
5. **Operator stop-condition check** — Supabase/Notion/GitHub writes; foreman/queue destructive ops; paid spend; public publish/deploy; deletion/move/overwrite. Standing rule: **the user's prior "go" applies ONLY to the scope I specified at that moment, not future similar ops.**
6. **Execute** — exact SQL/code from the gate. Read-back to verify.
7. **Audit** — INSERT `ai_audit` row (actor, action, target_table=NOT NULL, target_id, payload jsonb). Update queue contract (`completed_verified` / `partial_*` / `awaiting_operator_signoff`). Move done contracts from `pending/` to `done/`.

```bash
cat spawned_prompts/<task>_codex_gate.md | codex exec -s read-only - > audit/<task>/codex_gate_verdict.txt 2>&1
```

## 1. Problem

`Document Control Validation` workflow (added in PR #25) fails on every PR that does not contain real `workers/<name>/MANIFEST.json` files, because the bash `for` loops iterate the literal glob pattern when no files match:

```
for manifest in workers/*/MANIFEST.json; do
  ...
done
```

With default bash, the unmatched glob expands to the literal string `workers/*/MANIFEST.json`. The subsequent `python3 -c "...open('$manifest')..."` call fails with `FileNotFoundError`, the error is swallowed by `2>/dev/null`, and the script proceeds to its FAIL=1 branch — printing the literal-string filename as "missing required field".

Repo currently contains only `workers/_base/README.md`. No PR can pass DC compliance until either (a) the workflow is hardened, or (b) every PR seeds real worker manifests. (b) is impractical for any PR not specifically about workers, so (a) is the right fix.

- AC4: When the repo has a worker manifest missing a required field, step 2 still fails.
